# Event Run of Show

Generates a minute-by-minute event timeline with named owners for each block, plus prep and day-of checklists that catch the things that always get forgotten (mics, dietary needs, room reset, livestream test).

Built for the kind of events run constantly at a university: panels, info sessions, receptions, faculty retreats, board visits, capstone presentations, ITAC meetings, recruiting events, public talks.

---

## When to use

Triggers:

- "Run of show for [event]"
- "Help me plan [event]"
- "Build me an agenda for [event]"
- "Event timeline for [event]"
- User describes an upcoming event and asks for any kind of timeline / structure help

---

## Inputs to collect

Ask for whatever isn't already provided. Don't ask for everything at once — request the essentials, then fill in details.

### Essentials (ask up front)

1. **Event name and purpose** — What is this and why is it happening?
2. **Date and start/end time**
3. **Location** — room, building, virtual, hybrid
4. **Audience** — who's coming, how many, faculty/student/external mix
5. **Goal** — what does success look like at the end?

### Detail (ask once essentials are clear)

6. **Speakers / VIPs / honorees** — names and roles
7. **Format** — panel, lecture, networking, mixed
8. **Catering** — if any, and what type (full meal vs. light bites)
9. **Tech needs** — mics, recording, livestream, slides, captioning
10. **Special considerations** — dietary, accessibility, security, press, photography
11. **Owning team** — who's running the event; who else is helping

---

## When the invite isn't plannable yet

If the essentials (venue, audience, goal, format) aren't provided and can't be reasonably inferred — for example, the body says "more details coming," the location is a placeholder like "TBD" or "In-Person Event," or the format is genuinely unknown — **don't produce a speculative run of show.** Placeholders in this template get treated as facts when staff share the document. A run of show built on guesses is worse than no run of show.

Instead:

1. Briefly name what's known and what's missing.
2. List 2–3 focused questions to bring back to the organizer — not ten.
3. **If the user is an attendee rather than the organizer**, offer to produce an attendee-side prep brief once details arrive — what to bring, who to seek out, how to use the time. That's a different deliverable, and the right one for that role.
4. **Check past versions of recurring events** if available (e.g., past Teams meetings for a recurring group, prior calendar invites with the same series ID) — they often hold context the current invite is missing.

## Standard time blocks

For most events, structure includes:

| Block | Typical duration | Notes |
|-------|------------------|-------|
| **Setup** | 30–60 min before doors | A/V test, name tags, signage, catering setup |
| **Doors open / arrivals** | 15–30 min | Check-in, light networking, coffee |
| **Welcome** | 3–5 min | Lead introduces purpose, recognizes VIPs, sets expectations |
| **Main program** | 30–90 min | Panel / lecture / presentation core |
| **Q&A** | 10–20 min | Plan a "seed question" if Q&A is critical |
| **Closing remarks** | 3–5 min | Thanks, next steps, exit instructions |
| **Reception / networking** | 30–60 min | Optional; needed for relationship-building events |
| **Breakdown** | 30 min | Room reset, return equipment, retrieve A/V loaners |

Build the timeline backward from the start — setup tasks live at T-60, T-30, T-15.

---

## Standard prep checklist (adapt per event)

T-7 days:
- [ ] Confirm all speakers / VIPs (date, time, location, role)
- [ ] Confirm room booking and any A/V reservations
- [ ] Send calendar invites with full details to all internal staff
- [ ] Order catering (with dietary options) if applicable
- [ ] Order signage / name tags / printed agendas if applicable

T-3 days:
- [ ] Walk the room with A/V and confirm setup
- [ ] Send reminder + parking/wayfinding info to attendees
- [ ] Confirm catering headcount
- [ ] Brief any student volunteers

T-1 day:
- [ ] Print final attendee list, name tags, agendas
- [ ] Confirm with speakers (any arrival changes, slide check)
- [ ] Verify recording / livestream setup if applicable
- [ ] Charge any roving mics, clickers, laptops

Day of (morning):
- [ ] Pick up any pre-orders / loaner equipment
- [ ] Mic check, slides loaded and tested
- [ ] Catering arrival confirmed
- [ ] Signage placed, doors propped (or unlocked)
- [ ] Lead has all key contacts in phone

---

## Risks that always bite

Surface these in every brief — staff will recognize them:

- **A/V failure** — laptop won't connect, mic batteries dead, livestream black. Always have a backup laptop and spare batteries.
- **No-show speaker** — have a backup question or fill segment ready.
- **Late catering** — confirm delivery window; have a Plan B drink/snack vendor on speed-dial.
- **Wrong room** — confirm room number on every comms; post wayfinding signs.
- **Accessibility miss** — captioning not working, room not navigable; check 24 hours out.
- **Photography / consent** — confirm photo policy with speakers and post a notice if photographing the audience.
- **Overrun** — have a soft and a hard stop. Brief the moderator on time signals.
- **Building access after hours** — verify card access for all helpers; have a backup contact for facilities.

## Adapting the template

| Event type | Emphasis |
|------------|----------|
| **Panel discussion** | Pre-event speaker briefing block. Strong moderator timing. Q&A planned, not improvised. |
| **Info session (admissions / programs)** | Lead with audience needs. Build in 1:1 time at the end. Take-away material ready. |
| **Reception / networking** | Light formal program; emphasis on flow, food/drink replenishment, exit timing. |
| **Faculty retreat** | Build in real breaks (10 min every 90 min). Plan facilitation, not just speakers. |
| **Board / advisory meeting** | Tight timing, formal materials, reserved seating, dietary attention. |
| **Capstone presentations** | Strict per-team timing, transition blocks, a moderator clock. |
| **Hybrid event** | Build in extra A/V buffer. Assign a remote moderator. Plan for chat-based Q&A. |
| **Recurring event** | Reuse the template; only call out what's different this time. |

## Example

**User:**
> "Help me plan a run of show. We have a Batten Hour talk next Thursday May 7, 12:15pm in Garrett Hall room 100. Speaker is Dr. Rivera from the Miller Center talking about state-level AI policy. Light lunch served. Expect ~60 attendees, mix of students and faculty. We're recording for the website."

**Response:**

```
# Run of Show — Batten Hour: Dr. Rivera on State-Level AI Policy
Thursday, May 7, 2026 · 12:15–1:15 PM · Garrett Hall Room 100 · ~60 attendees

## Goal
A clear, well-attended Batten Hour talk that's properly recorded for later web publication — and a positive experience for Dr. Rivera that supports future Miller Center collaboration.

## Key contacts
- Lead: [you]
- Speaker: Dr. Rivera (Miller Center) — confirm arrival time
- A/V / Recording: Garrett Hall A/V — book by Friday
- Catering: [vendor TBD]
- Facilities: [building contact]

## Run of show
| Time | Duration | Block | Owner | Notes |
|------|----------|-------|-------|-------|
| 11:15 AM | 30 min | Setup — A/V test, room layout, signage | A/V + Lead | Mic check, recording start tested |
| 11:45 AM | 15 min | Catering arrives, lunch setup | Catering | Dietary labels visible |
| 11:55 AM | 5 min | Speaker arrival, brief, mic check | Lead | Walk Dr. Rivera to room, settle |
| 12:00 PM | 15 min | Doors open, lunch service, attendees seat | Lead + volunteers | Hand-out: agenda + speaker bio |
| 12:15 PM | 5 min | Welcome + intro of Dr. Rivera | Lead | Confirm recording is rolling |
| 12:20 PM | 30 min | Talk | Dr. Rivera | Time signal at 25 min |
| 12:50 PM | 20 min | Moderated Q&A | Moderator | Have 1 seed question ready |
| 1:10 PM | 5 min | Thank speaker, closing remarks | Lead | Mention website recording will post |
| 1:15 PM | 30 min | Breakdown, walk speaker out, room reset | Lead + A/V | Stop recording, return mics |

## Prep checklist
- [ ] T-5: Confirm Dr. Rivera arrival time and dietary needs
- [ ] T-5: Book Garrett A/V with recording
- [ ] T-5: Order catering (~65 portions, vegetarian + GF options)
- [ ] T-3: Send calendar reminder + parking info to attendees
- [ ] T-3: Print speaker bio, agenda, name tag
- [ ] T-1: Confirm catering headcount and delivery window
- [ ] T-1: Test recording workflow with A/V
- [ ] T-1: Brief moderator on time signals and seed question
- [ ] Day-of AM: Charge clicker, lavalier mic; verify backup laptop

## Day-of checklist
- [ ] Pick up signage, name tags, handouts
- [ ] Mic check (lavalier + roving for Q&A)
- [ ] Slides loaded and tested
- [ ] Catering arrived and dietary labels placed
- [ ] Recording confirmed running before welcome remarks
- [ ] Phone has Dr. Rivera, A/V, catering, building contact

## Risks / what to watch
- Recording silently fails — have A/V verify rolling at 12:15 sharp.
- Q&A goes flat — moderator has seed question to open.
- Lunch line backs up — start service at 12:00, not 12:10.
- Speaker overrun — moderator gives 5-min and 1-min signals; hard stop at 12:50 to protect Q&A.
- Room temp / A/C — check 30 min before; Garrett 100 runs warm with 60 people.
```

---

## What this does NOT do

- **Make reservations or send communications** — produces the plan; the user executes.
- **Replace the lead's judgment** — the lead always owns the timing call on the day of.
- **Invent vendors or contacts** — if a contact isn't provided, leave a placeholder for the user to fill in.

---

## Privacy

Attendee names, speaker details, and internal logistics are sensitive. Keep them in the response only.
